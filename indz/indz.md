# Індивідуальне навчально-дослідне завдання (ІНДЗ)

## Загальна інформація

**Назва курсу:** Технології проектування та розробки мережевих ресурсів

**Обсяг:** 20 годин самостійної роботи

**Формат:** Індивідуальний проєкт

**Термін виконання:** протягом семестру

**Здача:** посилання на GitHub репозиторій через Moodle


## Завдання ІНДЗ

Створити повноцінний вебзастосунок з використанням FastAPI, застосовуючи принципи Clean Architecture та сучасні мережеві технології. Проєкт має продемонструвати здатність самостійно проєктувати та розробляти backend систему з дотриманням сучасних практик веброзробки.

## Обов'язкові технічні вимоги

### Backend архітектура

- Clean Architecture з розділенням на три шари: API (Presentation), Services (Business Logic), Repositories (Data Access)
- Dependency Injection для управління залежностями
- Pydantic для валідації даних та створення DTO

### База даних

- SQLAlchemy 2.0 з async підтримкою
- Alembic для міграцій
- Repository Pattern для роботи з даними
- Мінімум 3-4 пов'язані таблиці

### Мережеві технології (обрати 1 з наступних)

- WebSocket для real-time комунікації
- Інтеграція зовнішнього REST API через httpx
- Background tasks з Celery та Redis
- Webhooks (прийом або відправка)

### Безпека

- JWT автентифікація
- Хешування паролів
- Валідація вхідних даних
- Обробка помилок з правильними HTTP статусами

### Код та тестування

- Type hints для функцій
- Unit тести для сервісів (мінімум 5 тестів)
- Дотримання PEP 8

### DevOps

- Dockerfile для застосунку
- Docker Compose (API + PostgreSQL)
- Конфігурація через .env файл


## Теми для проєктів

Оберіть одну тему або запропонуйте власну.

### 1. Система управління завданнями

**Функціонал:**
- CRUD операції для завдань та проєктів
- Пріоритети та статуси завдань
- Призначення завдань користувачам
- Фільтрація та пошук
- Dashboard з статистикою

**Мережева технологія:** WebSocket для real-time оновлення статусів

### 2. Блог-платформа

**Функціонал:**
- Публікація статей з тегами
- Система коментарів
- Реакції на статті (like, bookmark)
- Пошук по тегах
- Профілі авторів

**Мережева технологія:** WebSocket для real-time коментарів


### 3. API онлайн магазину

**Функціонал:**
- Каталог товарів з категоріями
- Кошик покупця
- Створення замовлень
- Історія замовлень
- Пошук товарів з фільтрами

**Мережева технологія:** Webhook для обробки платежів або Background tasks для email

### 4. Сервіс скорочення URL

**Функціонал:**
- Створення коротких посилань
- Редірект на оригінальний URL
- Аналітика переходів
- Custom alias для посилань
- Термін дії посилань

**Мережева технологія:** Інтеграція API для QR кодів або Redis кешування


### 5. Система бронювання

**Функціонал:**
- Управління ресурсами (кімнати, столики)
- Створення бронювань
- Перевірка доступності
- Календар бронювань
- Підтвердження бронювань

**Мережева технологія:** WebSocket для real-time доступності або Background tasks для нагадувань

### 6. Система моніторингу API

**Функціонал:**
- Додавання endpoints для моніторингу
- Перевірка доступності
- Логування response time
- Dashboard з метриками
- Історія інцидентів

**Мережева технологія:** Background tasks (Celery) для періодичних перевірок


## Критерії оцінювання (10 балів)

### 1. Архітектура та структура (2 бали)

- Правильне розділення на шари (API, Services, Repositories)
- Використання Dependency Injection
- Наявність DTO (Pydantic schemas)

### 2. Функціональність (2 бали)

- Всі основні CRUD операції працюють
- Бізнес-логіка реалізована коректно
- Валідація вхідних даних
- Обробка помилок

### 3. База даних (1 бал)

- Правильна інтеграція SQLAlchemy
- Міграції через Alembic
- Repository Pattern
- Адекватна схема БД (3-4 таблиці з зв'язками)

### 4. Мережеві технології (2 бали)

- Реалізовано одну з обраних технологій
- Функціонал працює коректно
- Обробка помилок та edge cases

### 5. Безпека та якість коду (1 бал)

- JWT автентифікація працює
- Type hints використовуються
- Код читабельний та структурований

### 6. Тестування (1 бал)

- Наявні unit тести для сервісів (мінімум 5)
- Тести проходять успішно
- Використання pytest

### 7. DevOps та документація (1 бал)

- Docker Compose працює (API + PostgreSQL)
- README.md повний та зрозумілий
- .env.example наявний
- Проєкт запускається без проблем

## Здача роботи

### Формат

Через Moodle подати **посилання на публічний GitHub репозиторій**.

Репозиторій має містити:
- Весь код проєкту
- README.md з повною інформацією
- Dockerfile та docker-compose.yml
- .env.example

### Перевірка

Викладач клонує репозиторій та запускає проєкт через Docker Compose. Проєкт має запуститись без помилок та відповідати всім технічним вимогам.
